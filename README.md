# PDF/HTML 閱讀地圖 (PDF Reading Map)

輕量級文件分析工具。設計目的是為了快速從大量的 PDF 文件（如服務建議書、政策白皮書）中提煉出關鍵需求，並產出結構化的追蹤表 (RTM)。https://progress01.github.io/PDFREADMAP/

### 為什麼需要這個工具？

* **效能優化**：採用分批載入技術，即使是數百頁的大型 PDF 也不會讓瀏覽器卡頓。
* **需求管理**：不只是畫重點，還能填寫 RTM ID、優先級與處理狀態。
* **資料隱私**：所有操作皆在您的電腦瀏覽器內完成，資料不會上傳至任何雲端伺服器。
* **交付方便**：可打包成「唯讀版」，方便將成果交付給主管查閱。

---

### 🚀 快速開始

1. **開啟工具**：直接雙擊 `PDF_Reading_Map.html` 檔案（建議使用 Chrome 或 Edge 瀏覽器）。
2. **匯入文件**：點擊左上角的 **「📂 匯入 PDF」**，選擇您要分析的 PDF 檔案。
3. **開始標註**：
* **手動標註**：用滑鼠選取文字，會跳出視窗填寫需求編號、標籤與筆記。


4. **分批閱讀**：透過下方的導覽列（上一批 / 下一批）切換閱讀範圍，每次顯示 10 頁以保持流暢。

---

### 📂 檔案匯出與交付說明

本工具支援三種匯出模式，請依用途選擇：

#### 1. 💾 存檔 (JSON) - **給自己備份**

* **用途**：暫停工作，下次繼續編輯。
* **內容**：只包含標註資料，**不含** PDF 原檔。
* **還原方式**：下次打開工具 -> 點「📥 載入專案」選擇 JSON -> 再點「📂 匯入 PDF」選擇原始文件。

#### 2. 📊 匯出 RTM 報表 (Excel) - **產出分析結果**

* **用途**：製作結案報告、需求追蹤矩陣。
* **內容**：包含 RTM 編號、頁碼、原文截錄、優先級、您的分析筆記。
* **特色**：已自動排版並上色，可直接複製貼上至正式文件。

#### 3. 📦 匯出閱讀版 HTML - **給老闆/客戶看**

* **用途**：交付成果，讓他人瀏覽您的標註（唯讀模式）。
* **操作流程**：
1. 點擊 **「📦 匯出閱讀版 HTML」**，下載產生的 HTML 檔。
2. 將此 **HTML 檔** 與 **原始 PDF** 一起寄給對方。
3. 對方打開 HTML 後，依指示選取 PDF 即可看到完整標註內容。



---

### 💡 使用小技巧

* **自動存檔**：
系統會自動將標註暫存在瀏覽器中。若不小心關閉視窗，重新開啟時系統會詢問是否還原草稿。
* **關於圖片與表格**：
此工具專注於「文字分析」。若 PDF 內含大量圖表，建議在 Excel 匯出後，再手動針對關鍵圖表進行截圖補充。

---

### ⚠️ 注意事項

1. **PDF 格式限制**：請使用標準文字版 PDF。若是掃描圖片檔（無法選取文字的 PDF），此工具無法讀取內容。
2. **瀏覽器建議**：請使用現代瀏覽器（Google Chrome, Microsoft Edge）。
3. **搜尋範圍**：使用 `Ctrl+F` 僅能搜尋「當前顯示批次」的頁面。若要搜尋整份文件，請務必使用工具列上的 **「🔍 全域搜尋」** 功能。
